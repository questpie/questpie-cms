# Admin Setup

Setting up the admin panel requires three steps: add the plugin to config, add the module, and set up the client builder.

## 1. Install

```bash
bun add @questpie/admin
```

## 2. Server Plugin

Add `adminPlugin()` to your runtime config:

```ts title="questpie.config.ts"
import { adminPlugin } from "@questpie/admin/server";
import { runtimeConfig } from "questpie";

export default runtimeConfig({
  plugins: [adminPlugin()],
  app: { url: process.env.APP_URL || "http://localhost:3000" },
  db: { url: process.env.DATABASE_URL },
  secret: process.env.APP_SECRET,
});
```

The plugin tells codegen to discover admin-specific file conventions: `sidebar.ts`, `dashboard.ts`, `branding.ts`, `blocks/`, `admin-locale.ts`.

## 3. Module

Add `adminModule` to your modules:

```ts title="modules.ts"
import { adminModule, auditModule } from "@questpie/admin/server";

export default [adminModule, auditModule] as const;
```

| Module | Provides |
|---|---|
| `adminModule` | User collection, auth pages, admin UI |
| `auditModule` | Audit log collection, timeline widget |

## 4. Admin Convention Files

Create the admin convention files:

```ts title="sidebar.ts"
import { sidebar } from "#questpie";

export default sidebar({
  sections: [
    { id: "overview", title: { en: "Overview" } },
    { id: "content", title: { en: "Content" } },
  ],
  items: [
    {
      sectionId: "overview",
      type: "link",
      label: { en: "Dashboard" },
      href: "/admin",
      icon: { type: "icon", props: { name: "ph:house" } },
    },
    {
      sectionId: "content",
      type: "collection",
      collection: "posts",
    },
  ],
});
```

```ts title="branding.ts"
import { branding } from "#questpie";

export default branding({
  name: { en: "My App Admin" },
});
```

## 5. Run Codegen

```bash
bunx questpie generate
```

## 6. Client Builder (qa)

Set up the admin client in your frontend:

```ts title="admin/builder.ts"
import { qa } from "@questpie/admin/client";
import type { AppConfig } from "@/questpie/server/.generated";

export const admin = qa<AppConfig>();
```

The `qa()` function is the admin client entrypoint. It creates a typed builder that gives you autocompletion for collections, globals, and admin APIs.

## 7. Mount the Admin

Add the admin routes to your app. The admin panel is typically served at `/admin`:

```ts title="routes/admin/$.tsx"
import { AdminRouter } from "@questpie/admin/client";
import { admin } from "@/questpie/admin/builder";

export default function AdminPage() {
  return <AdminRouter admin={admin} />;
}
```

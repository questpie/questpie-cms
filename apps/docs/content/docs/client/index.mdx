# Client SDK

The client stack helps you consume a QUESTPIE runtime contract from frontend and service clients. It is runtime-driven: types flow from your server schema and RPC definitions.

## Why This Exists

Your runtime exposes HTTP endpoints. The client SDK makes them ergonomic and type-safe: the same field and RPC types from the server propagate to queries and mutations without code generation.

## Mental Model

```text
Server Runtime (collections, globals, fields, RPC)
  │
  ├── Adapter (Hono/Elysia/Next)  → HTTP endpoints
  │     └── createFetchHandler()   → generic fetch handler
  │
  ├── questpie/client
  │     └── createClient()         → typed HTTP client
  │
  ├── Client SDK (@questpie/tanstack-query)
  │     ├── createQuestpieQueryOptions()  → typed query/mutation options
  │     └── realtime query options         → SSE-backed updates
  │
  └── OpenAPI (@questpie/openapi)
        └── generateOpenApiSpec()   → OpenAPI 3.1.0 spec + Scalar UI
```

## Quick Example

```ts
import { createClient } from "questpie/client";
import { createQuestpieQueryOptions } from "@questpie/tanstack-query";
import { useQuery, useMutation } from "@tanstack/react-query";
import type { AppCMS, AppRpc } from "@/questpie/server/cms";

const client = createClient<AppCMS, AppRpc>({
  baseURL: "http://localhost:3000",
  basePath: "/api/cms",
});

const cms = createQuestpieQueryOptions(client, {
  keyPrefix: "cms",
});

// Typed query
const { data } = useQuery(
  cms.collections.posts.find({
    where: { published: true },
    limit: 10,
  }),
);
// data is typed: { docs: Post[], totalDocs: number, ... }

// Typed mutation
const createPost = useMutation(cms.collections.posts.create());

// Typed RPC
const { data: stats } = useQuery(
  cms.rpc.getOverviewStats.query({}),
);
```

## Packages

| Package | Purpose |
|---|---|
| `questpie/client` | Typed runtime HTTP client |
| `@questpie/tanstack-query` | TanStack Query integration with typed proxies |
| `@questpie/hono` | Hono adapter (server middleware + client) |
| `@questpie/elysia` | Elysia adapter (plugin + Eden Treaty client) |
| `@questpie/next` | Next.js adapter (route handlers) |
| `@questpie/openapi` | OpenAPI 3.1.0 spec generation + Scalar docs UI |

`@questpie/admin` is documented separately in [Admin UI](/docs/admin). It is an optional interface layer, not required for client SDK usage.

# Field Types Overview

The admin module provides 22+ field types for building forms and tables. Each field type has a form component (for editing) and a cell component (for table display).

## Using Fields

Access fields through the registry proxy:

```typescript
.fields(({ r }) => ({
  title: r.text({ label: "Title" }),
  email: r.email({ label: "Email" }),
  status: r.select({ label: "Status", options: [...] }),
}))
```

## Field Categories

### Text Fields

| Field | Usage | Description |
|-------|-------|-------------|
| `text` | `r.text()` | Single-line text input |
| `email` | `r.email()` | Email with validation |
| `password` | `r.password()` | Masked password input |
| `textarea` | `r.textarea()` | Multi-line text |

```typescript
title: r.text({
  label: "Title",
  placeholder: "Enter title...",
  maxLength: 200,
}),
email: r.email({
  label: "Email",
  required: true,
}),
bio: r.textarea({
  label: "Biography",
  rows: 5,
}),
```

[View Text Fields Reference](/docs/admin/fields/text-fields)

### Numeric Fields

| Field | Usage | Description |
|-------|-------|-------------|
| `number` | `r.number()` | Numeric input with min/max |

```typescript
price: r.number({
  label: "Price",
  min: 0,
  max: 10000,
  step: 0.01,
}),
quantity: r.number({
  label: "Quantity",
  min: 0,
  integer: true,

```

### Selection Fields

| Field | Usage | Description |
|-------|-------|-------------|
| `select` | `r.select()` | Dropdown selection |
| `checkbox` | `r.checkbox()` | Boolean checkbox |
| `switch` | `r.switch()` | Toggle switch |

```typescript

  label: "Status",
  options: [
    { label: "Draft", value: "draft" },
    { label: "Published", value: "published" },

```

### Relation Fields

| Field | Usage | Description |
|-------|-------|-------------|
| `relation` | `r.relation()` | Link to other collections |
| `reverseRelation` | `r.reverseRelation()` | Show related items |

```typescript
author: r.relation({
  label: "Author",
  targetCollection: "users",
}),

  label: "Tags",
  targetCollection: "tags",
  type: "multiple",
  orderable: true,
}),

  label: "Posts",
  targetCollection: "posts",
  foreignKey: "authorId",
}),
```

[View Relation Fields Reference](/docs/admin/fields/relation-fields)

### JSON & Structured Fields

| Field | Usage | Description |
|-------|-------|-------------|
| `json` | `r.json()` | Raw JSON editor |
| `object` | `r.object()` | Typed object with fields |
| `array` | `r.array()` | Repeatable items |

```typescript
metadata: r.json({
  label: "Metadata",
}),
address: r.object({
  label: "Address",
  fields: ({ r }) => ({
    street: r.text({ label: "Street" }),
    city: r.text({ label: "City" }),
    zip: r.text({ label: "ZIP" }),
  }),
}),
socialLinks: r.array({
  label: "Social Links",
  item: ({ r }) => ({
    platform: r.select({ label: "Platform", options: [...] }),
    url: r.text({ label: "URL" }),
  }),
}),
```

## Common Options

All field types support these common options:

| Option | Type | Description |
|--------|------|-------------|
| `label` | `string \| I18nText` | Display label |
| `description` | `string \| I18nText` | Help text |
| `placeholder` | `string \| I18nText` | Placeholder text |
| `required` | `boolean \| (values) => boolean` | Required validation |
| `readOnly` | `boolean \| (values) => boolean` | Disable editing |
| `visible` | `(values) => boolean` | Conditional visibility |
| `defaultValue` | `any` | Default for new items |

### Conditional Logic

```typescript
status: r.select({
  label: "Status",
  options: [
    { label: "Active", value: "active" },
    { label: "Cancelled", value: "cancelled" },
  ],
}),
// Show only when cancelled
cancellationReason: r.textarea({
  label: "Reason",
  visible: (values) => values.status === "cancelled",
  required: (values) => values.status === "cancelled",
}),
// Readonly when published
publishedAt: r.datetime({
  label: "Published",
  readOnly: (values) => values.status === "published",
}),
```

## Localized Labels

All text options support i18n:

```typescript
title: r.text({
  label: { en: "Title", sk: "Titulok" },
  placeholder: { en: "Enter title...", sk: "Zadajte titulok..." },
  description: { en: "The main headline", sk: "Hlavny nadpis" },
}),
```

## Localized Content

For fields that store localized content (separate value per locale):

```typescript
title: r.text({
  label: "Title",
  localized: true,  // Creates per-locale values
}),
```

When `localized: true`, the admin UI shows a locale switcher and stores values separately for each configured locale.

## Field Presets

Create reusable field configurations:

```typescript
// admin/field-presets.ts
export const titleField = (label = "Title") => r.text({
  label,
  maxLength: 200,
  required: true,
});

export const statusField = () => r.select({
  label: "Status",
  options: [
    { label: "Draft", value: "draft" },
    { label: "Published", value: "published" },
  ],
  defaultValue: "draft",
});

// Usage
.fields(({ r }) => ({
  title: titleField("Post Title"),
  status: statusField(),
}))
```

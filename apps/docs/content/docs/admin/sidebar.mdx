# Sidebar Configuration

The sidebar builder creates the navigation structure for your admin interface.

## Creating a Sidebar Config

```typescript
import { qa } from "@questpie/admin/client";
import {
  HouseIcon,
  FileTextIcon,
  UsersIcon,
  GearIcon,
} from "@phosphor-icons/react";
import type { AppCMS } from "../server/cms";

export const sidebarConfig = qa
  .sidebar<AppCMS>()
  .section("main", (s) => s.items([...]))
  .section("content", (s) => s.title("Content").items([...]))
  .section("settings", (s) => s.title("Settings").items([...]));
```

The generic `<AppCMS>` provides autocomplete for collection and global names.

## Sections

Sections group navigation items. Each section can have:

- **id** - Unique identifier (first argument to `.section()`)
- **title** - Optional header text
- **icon** - Optional section icon
- **items** - Navigation items

```typescript
.section("content", (s) =>
  s
    .title({ en: "Content", sk: "Obsah" })
    .icon(FileTextIcon)
    .items([
      { type: "collection", collection: "posts" },
      { type: "collection", collection: "pages" },
    ])
)
```

### Section Without Title

For top-level items without a group header:

```typescript
.section("main", (s) =>
  s.items([
    { type: "link", label: "Dashboard", href: "/admin", icon: HouseIcon },
  ])
)
```

## Item Types

### Collection Item

Link to a collection list:

```typescript
{
  type: "collection",
  collection: "posts",           // Backend collection name (autocomplete!)
  icon: FileTextIcon,            // Optional override
  label: "Blog Posts",           // Optional override (uses collection meta.label by default)
}
```

### Global Item

Link to a global edit page:

```typescript
{
  type: "global",
  global: "siteSettings",        // Backend global name (autocomplete!)
  icon: GearIcon,
  label: "Site Settings",        // Optional override
}
```

### Link Item

Custom navigation link:

```typescript
{
  type: "link",
  label: { en: "Dashboard", sk: "Prehlad" },
  href: "/admin",
  icon: HouseIcon,
  external: false,               // Open in new tab
}
```

### Divider

Visual separator:

```typescript
{ type: "divider" }
```

## Complete Example

```typescript
// admin/sidebar.ts
import { qa } from "@questpie/admin/client";
import {
  CalendarCheckIcon,
  FileTextIcon,
  GearIcon,
  HouseIcon,
  ScissorsIcon,
  StarIcon,
  UsersIcon,
} from "@phosphor-icons/react";
import type { AppCMS } from "../server/cms";

export const sidebarConfig = qa
  .sidebar<AppCMS>()
  // Dashboard link (no section title)
  .section("main", (s) =>
    s.items([
      {
        type: "link",
        label: { en: "Dashboard", sk: "Prehlad" },
        href: "/admin",
        icon: HouseIcon,
      },
    ])
  )
  // Content section
  .section("content", (s) =>
    s
      .title({ en: "Content", sk: "Obsah" })
      .icon(FileTextIcon)
      .items([
        { type: "collection", collection: "pages", icon: FileTextIcon },
      ])
  )
  // Bookings section
  .section("bookings", (s) =>
    s
      .title({ en: "Bookings", sk: "Rezervacie" })
      .icon(CalendarCheckIcon)
      .items([
        { type: "collection", collection: "appointments", icon: CalendarCheckIcon },
        { type: "collection", collection: "reviews", icon: StarIcon },
      ])
  )
  // Staff section
  .section("staff", (s) =>
    s
      .title({ en: "Staff & Services", sk: "Personal a sluzby" })
      .icon(UsersIcon)
      .items([
        { type: "collection", collection: "barbers", icon: UsersIcon },
        { type: "collection", collection: "services", icon: ScissorsIcon },
      ])
  )
  // Settings section
  .section("settings", (s) =>
    s
      .title({ en: "Settings", sk: "Nastavenia" })
      .icon(GearIcon)
      .items([
        { type: "global", global: "siteSettings", icon: GearIcon },
        { type: "divider" },
        {
          type: "link",
          label: "Documentation",
          href: "https://questpie.dev/docs",
          icon: FileTextIcon,
          external: true,
        },
      ])
  );
```

## Typed Sidebar Items

For standalone item arrays with type safety:

```typescript
import type { SidebarItemForApp } from "@questpie/admin/client";

const contentItems: SidebarItemForApp<AppCMS>[] = [
  { type: "collection", collection: "posts" },
  { type: "collection", collection: "pages" },
];
```

## Adding to Admin

Register the sidebar in your main admin config:

```typescript
// admin/admin.ts
import { sidebarConfig } from "./sidebar";

export const admin = qa()
  .use(adminModule)
  .sidebar(sidebarConfig);
```

## Conditional Items

Show items based on user role or permissions:

```typescript
// In a component, use hooks to conditionally render
import { useCurrentUser, useHasRole } from "@questpie/admin/client";

function ConditionalNav() {
  const hasAdminRole = useHasRole("admin");

  // Sidebar items can be filtered based on roles
  // The admin framework handles this automatically when
  // you configure access control on collections
}
```

## Icons

Always use React component icons, not strings:

```typescript
// Correct
import { FileTextIcon } from "@phosphor-icons/react";
{ type: "collection", collection: "posts", icon: FileTextIcon }

// Incorrect - will not work
{ type: "collection", collection: "posts", icon: "FileTextIcon" }
```

Recommended icon library: `@phosphor-icons/react`

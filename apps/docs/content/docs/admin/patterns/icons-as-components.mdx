# Icons as Components

In QUESTPIE Admin, **icons must be React components**, not strings. This ensures type safety and allows for consistent icon rendering.

## The Rule

```typescript
import { Newspaper, Users, Settings } from "lucide-react";

// ✅ Correct - Component reference
.meta({ icon: Newspaper })

// ❌ Wrong - String
.meta({ icon: "Newspaper" })

// ❌ Wrong - JSX element
.meta({ icon: <Newspaper /> })
```

## Why Components?

### 1. Type Safety

TypeScript can verify the icon is a valid component:

```typescript
import { IconType } from "lucide-react";

interface MetaConfig {
  icon?: IconType;  // Only accepts component
}

// TypeScript catches errors
.meta({ icon: "NotAComponent" })  // ❌ Type error!
```

### 2. Flexible Rendering

Components can be rendered with different props:

```tsx
// Admin can render with appropriate size/color
function SidebarItem({ icon: Icon, label }) {
  return (
    <div>
      <Icon className="h-4 w-4" />  {/* Render with props */}
      <span>{label}</span>
    </div>
  );
}
```

### 3. Tree Shaking

Only imported icons are included in the bundle:

```typescript
// Only Newspaper is bundled
import { Newspaper } from "lucide-react";

.meta({ icon: Newspaper })
```

## Usage Examples

### Collection Icons

```typescript
import { Newspaper, Users, Image, Tag } from "lucide-react";

const postsAdmin = qab.collection("posts")
  .meta({
    icon: Newspaper,
    label: "Posts",
  });

const usersAdmin = qab.collection("users")
  .meta({
    icon: Users,
    label: "Users",
  });

const mediaAdmin = qab.collection("media")
  .meta({
    icon: Image,
    label: "Media",
  });

const tagsAdmin = qab.collection("tags")
  .meta({
    icon: Tag,
    label: "Tags",
  });
```

### Sidebar Icons

```typescript
import {
  LayoutDashboard,
  FileText,
  Users,
  Settings,
  Folder,
} from "lucide-react";

qa.sidebar(({ s }) => [
  s.link({
    label: "Dashboard",
    icon: LayoutDashboard,
    href: "/admin",
  }),
  s.section({
    label: "Content",
    icon: Folder,
    items: [
      s.collection({ collection: "posts", icon: FileText }),
      s.collection({ collection: "users", icon: Users }),
    ],
  }),
  s.global({
    global: "settings",
    icon: Settings,
  }),
]);
```

### Dashboard Widget Icons

```typescript
import { FileText, Users, Eye, TrendingUp } from "lucide-react";

.dashboard(({ w }) => ({
  widgets: [
    w.stats({
      label: "Posts",
      icon: FileText,
      value: async () => postsCount,
    }),
    w.stats({
      label: "Users",
      icon: Users,
      value: async () => usersCount,
    }),
  ],
}))
```

## Icon Libraries

QUESTPIE Admin works with any icon library that exports React components:

### Lucide React (Recommended)

```typescript
import { Home, Settings, User } from "lucide-react";
```

### Heroicons

```typescript
import { HomeIcon, CogIcon, UserIcon } from "@heroicons/react/24/outline";
```

### Phosphor Icons

```typescript
import { House, Gear, User } from "@phosphor-icons/react";
```

### React Icons

```typescript
import { FiHome, FiSettings, FiUser } from "react-icons/fi";
```

## Custom Icons

You can create custom icon components:

```tsx
// Custom SVG icon
function CustomIcon(props: React.SVGProps<SVGSVGElement>) {
  return (
    <svg
      xmlns="http://www.w3.org/2000/svg"
      viewBox="0 0 24 24"
      fill="none"
      stroke="currentColor"
      strokeWidth={2}
      {...props}
    >
      <path d="M12 2L2 7l10 5 10-5-10-5z" />
      <path d="M2 17l10 5 10-5" />
      <path d="M2 12l10 5 10-5" />
    </svg>
  );
}

// Use in config
.meta({ icon: CustomIcon })
```

## Common Mistakes

### ❌ String Icons

```typescript
// Doesn't work - strings can't be rendered
.meta({ icon: "Newspaper" })
```

### ❌ JSX Elements

```typescript
// Doesn't work - can't pass props to JSX
.meta({ icon: <Newspaper size={16} /> })
```

### ❌ Inline Components

```typescript
// Works but bad for performance (creates new component each render)
.meta({ icon: () => <Newspaper /> })

// Better - use imported component
.meta({ icon: Newspaper })
```

### ❌ Wrong Import

```typescript
// Wrong - importing from wrong path
import Newspaper from "lucide-react/dist/esm/icons/newspaper";

// Correct
import { Newspaper } from "lucide-react";
```

## TypeScript Integration

Define icon types for your collections:

```typescript
import type { LucideIcon } from "lucide-react";

interface CollectionMeta {
  label: string;
  labelPlural?: string;
  icon?: LucideIcon;
  description?: string;
}
```

## Best Practices

1. **Import at the top** - Keep icon imports organized
2. **Use consistent library** - Stick to one icon library
3. **Match visual style** - Use icons with similar stroke weight
4. **Be semantic** - Choose icons that represent the content

```typescript
// Good - imports at top, consistent library
import {
  Newspaper,
  Users,
  Settings,
  Image
} from "lucide-react";

// Bad - mixed libraries
import { Newspaper } from "lucide-react";
import { FiUsers } from "react-icons/fi";
import { CogIcon } from "@heroicons/react/24/outline";
```

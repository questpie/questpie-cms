# File Convention

QuestPie uses your file system as the source of truth. Drop a file in the right directory, run codegen, and it's part of your app. No manual registration.

## Directory Categories

Each directory maps to a category of entity:

| Directory | Entity | Export Style | Key Derivation |
|---|---|---|---|
| `collections/` | Collections | Default or named | Filename в†’ camelCase |
| `globals/` | Globals | Default or named | Filename в†’ camelCase |
| `routes/` | Routes | Default | Filename в†’ camelCase/slash path |
| `jobs/` | Jobs | Default | Filename в†’ camelCase |
| `routes/` (raw) | Routes | Default | Filename в†’ slash-separated path |
| `services/` | Services | Default | Filename в†’ camelCase |
| `emails/` | Email templates | Default | Filename в†’ camelCase |
| `blocks/` | Blocks | Named exports | Export name |
| `messages/` | i18n messages | Default | Filename в†’ locale key |
| `migrations/` | DB migrations | Default | Array (ordered) |
| `seeds/` | DB seeds | Default | Array (ordered) |

### Name derivation

Filenames are converted from kebab-case to camelCase:

```text
blog-posts.ts      в†’ blogPosts
create-booking.ts  в†’ createBooking
site-settings.ts   в†’ siteSettings
```

## Single-File Conventions

Some configs are single files, not directories:

| File | Factory | Purpose |
|---|---|---|
| `questpie.config.ts` | `runtimeConfig({...})` | DB, plugins, adapters |
| `modules.ts` | `export default [...]` | Module dependencies |
| `auth.ts` | `satisfies AuthConfig` | Auth configuration |
| `locale.ts` | `locale({...})` | Content locales |
| `hooks.ts` | `export default { collections, globals }` | Global lifecycle hooks (applied across all entities) |
| `access.ts` | `export default { read, create, update, delete }` | Default access rules for all entities |
| `context.ts` | `export default contextResolver(...)` | Custom context resolution (e.g., tenant scoping) |
| `fields.ts` | `export default { ...customFields }` | Custom field type definitions |
| `sidebar.ts` | `sidebar({...})` | Admin sidebar |
| `dashboard.ts` | `dashboard({...})` | Admin dashboard |
| `branding.ts` | `branding({...})` | Admin branding |
| `admin-locale.ts` | `adminLocale({...})` | Admin UI locale |

## Discovery Process

1. **Scan** вЂ” Codegen walks the configured directories
2. **Match** вЂ” Files matching the category pattern are picked up
3. **Import** вЂ” Each file is imported and its exports are read
4. **Key derivation** вЂ” Filename в†’ camelCase (or export name for named exports)
5. **Merge** вЂ” Project entities + module entities are merged
6. **Emit** вЂ” Generated types and runtime wiring in `.generated/`

## The `#questpie` Import

Collection and global files use `#questpie` as an import alias:

```ts
import { collection } from "#questpie";
```

This resolves to the generated app context, giving the builders access to field types and relation targets for autocompletion. It's configured via TypeScript path mapping.

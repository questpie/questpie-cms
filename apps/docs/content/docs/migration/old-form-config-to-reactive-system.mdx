# Old Form Config -> Reactive System

## New reactive model

Use `meta.admin` handlers on fields:

- `hidden`
- `readOnly`
- `disabled`
- `compute`
- dynamic `options`

## Example

```ts
slug: f.text({
  required: true,
  input: "optional",
  meta: {
    admin: {
      compute: {
        handler: ({ data }) => {
          const t = data.title;
          if (!t || typeof t !== "string") return undefined;
          return t
            .toLowerCase()
            .replace(/[^a-z0-9]+/g, "-")
            .replace(/(^-|-$)/g, "");
        },
        deps: ({ data }) => [data.title],
        debounce: 300,
      },
    },
  },
});
```

## Migration checklist

1. move conditional UI logic from client form hacks to server field meta
2. keep expensive dynamic option lookups server-side (`ctx.db`)
3. test behavior through introspection-driven form rendering

# Installation

QUESTPIE installation is runtime-first: install the backend baseline, choose one HTTP adapter, then add optional packages like admin UI, TanStack Query, or OpenAPI.

## Prerequisites

| Requirement | Version | Notes |
|---|---|---|
| **Bun** | 1.3+ | Primary runtime and package manager |
| **Node.js** | 20+ | Alternative to Bun (npm/pnpm also work) |
| **TypeScript** | 5.0+ | Required for type inference |
| **PostgreSQL** | 14+ | Primary database (PGlite supported for development) |

## Required Baseline Packages

Every QUESTPIE project starts with this runtime baseline:

```bash
bun add questpie zod drizzle-orm postgres
```

| Package | Purpose |
|---|---|
| `questpie` | Core platform: builders, fields, CRUD, hooks, access, RPC |
| `zod` | Validation schemas for RPC inputs and field validation |
| `drizzle-orm` | ORM used internally (also used for custom indexes and queries) |
| `postgres` | PostgreSQL driver for runtime database connections |

## Required Framework Adapter

Add exactly one adapter that matches your web framework:

```bash
# Hono (recommended for new projects)
bun add @questpie/hono hono

# Elysia
bun add @questpie/elysia elysia

# Next.js
bun add @questpie/next next
```

You only need **one** adapter. It creates the fetch handler that turns your backend runtime into HTTP endpoints.

## Optional Admin Interface

If you want the generated admin interface, add `@questpie/admin`:

```bash
bun add @questpie/admin
```

Usage notes:

- Server side: import `adminModule` from `@questpie/admin/server` and register it with `q.use(adminModule)`.
- Client side: import `qa` and `adminModule` from `@questpie/admin/client` to configure registries and rendering.
- Runtime-only path: skip this package and keep only your server runtime + API handlers.
- Admin login requires auth to be configured on the runtime (`.auth({ ... })`).

## Client SDK (Optional)

For frontend data fetching with type safety:

```bash
bun add @questpie/tanstack-query @tanstack/react-query
```

## OpenAPI (Optional)

For auto-generated API documentation:

```bash
bun add @questpie/openapi
```

## Local Database Alternative

For local development without a PostgreSQL server:

```bash
bun add @electric-sql/pglite
```

## Environment Variables

Create a `.env` file in your project root:

```bash
# Required
DATABASE_URL=postgres://postgres:postgres@localhost:5432/mydb
APP_URL=http://localhost:3000

# Required when using auth/admin login
BETTER_AUTH_SECRET=replace-me-in-production

# Optional: storage, email, etc.
# SMTP_HOST=localhost
# SMTP_PORT=1025
```

## Verify Installation

Create a simple test file to confirm the baseline runtime resolves:

```ts
import { q } from "questpie";

// If this compiles, you're ready
const builder = q({ name: "test" });
```

If you're using the admin interface, also verify these imports resolve:

```ts
import { adminModule } from "@questpie/admin/server";
import { qa } from "@questpie/admin/client";

void adminModule;
void qa;
```

Run with `bun run --bun your-file.ts` or `npx tsx your-file.ts` to verify.

## Common Mistakes

- **Missing `drizzle-orm` or `postgres`** — QUESTPIE depends on Drizzle internals and a PostgreSQL driver in the baseline runtime.
- **Installing admin too early** — Start from runtime baseline, then add `@questpie/admin` only when you need an internal UI.
- **Using wrong admin subpath** — `@questpie/admin/server` is for builder augmentation and `@questpie/admin/client` is for interface rendering.
- **Wrong PostgreSQL version** — QUESTPIE uses features like `jsonb`, full-text search, and `pg_trgm` that require PostgreSQL 14+.

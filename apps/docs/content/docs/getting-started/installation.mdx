# Installation

QUESTPIE requires a Node.js-compatible runtime, TypeScript, and PostgreSQL. This page covers everything you need to install before building your first CMS.

## Prerequisites

| Requirement | Version | Notes |
|---|---|---|
| **Bun** | 1.3+ | Primary runtime and package manager |
| **Node.js** | 20+ | Alternative to Bun (npm/pnpm also work) |
| **TypeScript** | 5.0+ | Required for type inference |
| **PostgreSQL** | 14+ | Primary database (PGlite supported for development) |

## Core Packages

Every QUESTPIE project needs these:

```bash
bun add questpie @questpie/admin zod drizzle-orm
```

| Package | Purpose |
|---|---|
| `questpie` | Core framework: builders, fields, CRUD, hooks, access, RPC |
| `@questpie/admin` | Admin module: server augmentation + client UI |
| `zod` | Validation schemas for RPC inputs and field validation |
| `drizzle-orm` | ORM used internally (you need it for custom indexes and queries) |

## Framework Adapter

Choose the adapter that matches your web framework:

```bash
# Hono (recommended for new projects)
bun add @questpie/hono hono

# Elysia
bun add @questpie/elysia elysia

# Next.js
bun add @questpie/next next
```

You only need **one** adapter. It creates the fetch handler that turns your CMS into HTTP endpoints.

## Client SDK (Optional)

For frontend data fetching with type safety:

```bash
bun add @questpie/tanstack-query @tanstack/react-query
```

## OpenAPI (Optional)

For auto-generated API documentation:

```bash
bun add @questpie/openapi
```

## Database Driver

```bash
bun add drizzle-orm postgres
```

For local development without a PostgreSQL server:

```bash
bun add @electric-sql/pglite
```

## Environment Variables

Create a `.env` file in your project root:

```bash
# Required
DATABASE_URL=postgres://postgres:postgres@localhost:5432/mydb
APP_URL=http://localhost:3000

# Auth (required for admin login)
BETTER_AUTH_SECRET=replace-me-in-production

# Optional: storage, email, etc.
# SMTP_HOST=localhost
# SMTP_PORT=1025
```

## Verify Installation

Create a simple test file to confirm everything resolves:

```ts
import { q, rpc, createFetchHandler } from "questpie";
import { adminModule } from "@questpie/admin/server";
import { qa } from "@questpie/admin/client";

// If this compiles, you're ready
const builder = q({ name: "test" }).use(adminModule);
```

Run with `bun run --bun your-file.ts` or `npx tsx your-file.ts` to verify.

## Common Mistakes

- **Missing `drizzle-orm`** — QUESTPIE uses Drizzle internally for schema generation. You need it installed even if you don't write Drizzle code directly.
- **Forgetting the server module** — `@questpie/admin` exports from two subpaths: `@questpie/admin/server` (for the builder) and `@questpie/admin/client` (for the UI). You need both.
- **Wrong PostgreSQL version** — QUESTPIE uses features like `jsonb`, full-text search, and `pg_trgm` that require PostgreSQL 14+.

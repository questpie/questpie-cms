# QUESTPIE Documentation

**v1 Beta** — QUESTPIE is a server-first TypeScript backend framework. The core `questpie` package is a fully standalone backend — collections, fields, relations, hooks, access control, validation, jobs, RPC, and a generated REST API. No admin panel required.

`@questpie/admin` is a separate module that reads your server schema and generates a complete admin panel — dashboard, table views, form editors, sidebar navigation — all from the definitions you already wrote.

## Two Packages, One Schema

```text
questpie            — standalone backend framework
                      collections, globals, fields, hooks, access, RPC
                      REST API, typed SDK, realtime (SSE)

@questpie/admin     — admin panel module (optional)
                      reads your server schema via introspection
                      generates dashboard, views, forms, sidebar
```

## Field Definitions

Fields are defined at the server level using the field builder `f`. Every field type maps to a database column, a REST API property, a validation rule, and (if admin is installed) a form input:

```ts
qb.collection("posts").fields((f) => ({
  title: f.text({ required: true, maxLength: 255 }),
  body: f.richText({ localized: true }),
  cover: f.upload({ to: "assets", mimeTypes: ["image/*"] }),
  author: f.relation({ to: "users" }),
  status: f.select({ options: ["draft", "published"] }),
  publishedAt: f.date(),
  tags: f.array({ of: f.text() }),
  metadata: f.object({
    fields: () => ({
      seoTitle: f.text(),
      seoDescription: f.textarea(),
    }),
  }),
}))
```

Available field types: `text`, `textarea`, `richText`, `number`, `boolean`, `date`, `time`, `email`, `url`, `select`, `multiSelect`, `upload`, `relation`, `object`, `array`, `json`, and more.

## Architecture at a Glance

```text
Server (questpie + admin server module)
  ├── collections / globals / fields     — schema + DB tables
  ├── hooks + access + validation        — behavior + business logic
  ├── RPC + jobs                         — custom endpoints + background tasks
  ├── realtime (SSE streaming)           — live updates
  ├── .admin() .list() .form()           — admin UI blueprint (optional)
  ├── .sidebar() .dashboard()            — navigation + widgets (optional)
  └── introspection contract             — server → admin bridge

Client (@questpie/admin)
  ├── field registry                     — how each field renders
  ├── list / edit view registry          — how views render
  ├── component registry                 — icons, badges, custom elements
  └── branding + locale + UI overrides   — look and feel
```

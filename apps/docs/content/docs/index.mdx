# QUESTPIE Documentation

<Callout type="warn">
**Active Development** — QUESTPIE is a bootstrapped, community-driven framework under active development. The API may still change between releases, but we follow semantic versioning to communicate breaking changes clearly. We do not recommend it for mission-critical production workloads yet. Full API stability is targeted for **v3**.
</Callout>

**v2** — QUESTPIE is a runtime-first TypeScript backend framework. Start with `questpie` to define schema, access rules, hooks, RPC, jobs, and HTTP endpoints. That runtime is fully usable on its own.

`@questpie/admin` is an optional module that reads your server schema via introspection and generates an admin panel. You can add it when you need an internal interface, or skip it and ship API-only services.

## Runtime First

```text
Required
questpie            — backend runtime (collections, fields, hooks, access, RPC)
@questpie/<adapter> — HTTP adapter (Hono, Elysia, or Next.js)

Optional
@questpie/admin     — generated admin interface via introspection
@questpie/tanstack-query, @questpie/openapi, other client tooling
```

The runtime contract comes first. Admin, SDKs, and OpenAPI all consume the same server definitions instead of redefining schema in multiple places.

## Field Definitions

Fields are defined on the runtime using the `f` field builder. Each field maps to database behavior, API validation, and query typing. If admin is installed, those same fields also drive form rendering.

```ts
q.collection("posts").fields((f) => ({
  title: f.text({ required: true, maxLength: 255 }),
  body: f.richText({ localized: true }),
  cover: f.upload({ to: "assets", mimeTypes: ["image/*"] }),
  author: f.relation({ to: "users" }),
  status: f.select({ options: ["draft", "published"] }),
  publishedAt: f.date(),
  tags: f.array({ of: f.text() }),
  metadata: f.object({
    fields: () => ({
      seoTitle: f.text(),
      seoDescription: f.textarea(),
    }),
  }),
}))
```

## Architecture at a Glance

```text
Runtime (questpie + one adapter)
  ├── collections / globals / fields   — schema + persistence contract
  ├── hooks + access + validation      — behavior + business rules
  ├── RPC + jobs                       — custom endpoints + background tasks
  ├── REST + realtime endpoints        — integration surface
  └── introspection contract           — consumed by optional tooling

Optional Interface Layer (@questpie/admin)
  ├── server module                    — .admin() .list() .form() .sidebar() .dashboard()
  └── client module                    — field/view/component registries
```

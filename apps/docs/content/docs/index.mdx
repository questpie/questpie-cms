# QUESTPIE CMS

**Headless, extensible CMS framework for TypeScript**

QUESTPIE CMS provides a type-safe backend with a beautiful, config-driven admin interface. Build content management systems without writing boilerplate.

## Key Features

- **Type-Safe Builder Pattern** - Full TypeScript inference from schema to API
- **Config-Driven Admin** - Beautiful UI generated from configuration
- **22+ Field Types** - Text, relations, uploads, rich text, blocks, and more
- **Page Builder** - Visual content editing with custom blocks
- **Framework Agnostic** - Works with Hono, Elysia, Next.js
- **i18n Ready** - Built-in localization for content and UI

## Quick Start

```bash
bun add questpie @questpie/admin @questpie/hono
```

```typescript
// Backend - Define your data
import { q, text, boolean } from "questpie";
import { starterModule } from "questpie/server";

const cms = q({ name: "my-cms" })
  .use(starterModule)
  .collections({
    posts: q.collection("posts")
      .fields({
        title: text("title").notNull(),
        content: text("content"),
        published: boolean("published").default(false),
      }),
  })
  .build({ db: { url: process.env.DATABASE_URL } });

export type AppCMS = typeof cms;
```

```typescript
// Admin UI - Configure the interface
import { qa, adminModule } from "@questpie/admin/client";
import type { AppCMS } from "./cms";

const admin = qa<AppCMS>()
  .use(adminModule)
  .collections({
    posts: qa.collection("posts")
      .fields(({ r }) => ({
        title: r.text({ label: "Title" }),
        content: r.richText({ label: "Content" }),
        published: r.switch({ label: "Published" }),
      })),
  });
```

## Architecture

QUESTPIE separates **backend** (data) from **admin** (UI):

```
┌─────────────────────────────────────────────────────────────┐
│                      Your Application                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────────────┐     ┌─────────────────────┐       │
│  │   Backend (q)       │     │   Admin UI (qa)     │       │
│  │   questpie          │     │   @questpie/admin   │       │
│  ├─────────────────────┤     ├─────────────────────┤       │
│  │ • Collections       │────>│ • Field configs     │       │
│  │ • Validation        │     │ • Table views       │       │
│  │ • Hooks             │     │ • Form views        │       │
│  │ • Access control    │     │ • Dashboard         │       │
│  │ • Authentication    │     │ • Sidebar           │       │
│  └─────────────────────┘     └─────────────────────┘       │
│           │                                                  │
│           │                                                  │
│  ┌─────────────────────────────────────────────────┐       │
│  │              Database (PostgreSQL)               │       │
│  └─────────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────────┘
```

**Backend (`questpie`)**: Handles data, validation, business logic, and authentication. Runs on the server.

**Admin UI (`@questpie/admin`)**: Provides the user interface for content management. Configures how data is displayed and edited.

Types flow from backend to admin, ensuring end-to-end type safety.

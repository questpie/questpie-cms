# QUESTPIE CMS

QUESTPIE CMS is a headless Content Management System engineered for modern TypeScript applications. It combines the flexibility of a code-first schema definition with the performance of a compiled runtime, providing developers with end-to-end type safety without the need for code generation.

## At a Glance

QUESTPIE CMS defines content structures ("Collections") through code. This approach ensures that your database schema and application types are always in sync.

```typescript
import { defineCollection, createCMS } from '@questpie/cms'
import { text, integer } from 'drizzle-orm/pg-core'

// 1. Define your content model
const products = defineCollection('products')
  .fields({
    name: text('name').notNull(),
    price: integer('price').notNull(),
    description: text('description'),
  })
  .access({
    read: () => true,
    create: 'admin',
  })

// 2. Initialize the CMS
const cms = createCMS({
  collections: [products],
  db: drizzle(process.env.DATABASE_URL!),
})

// 3. Query with full type safety
const product = await cms.crud('products', ctx).findFirst({
  where: { name: 'QUESTPIE Pro' }
})
```
